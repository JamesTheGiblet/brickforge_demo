## ğŸ—ï¸ How It Works (The Architecture)

The BlockForge ecosystem consists of two parts. This repository is **Part 2**.

```mermaid
graph LR
    A[User Input] -->|STL / Image / Text| B(BlockForge Engine);
    B -->|Voxelize & Optimize| C{JSON / Geometry};
    C -->|Import| D[This Web Suite];
    D -->|Paint & Configure| E[Final Order / PDF];
    style B fill:#f9f,stroke:#333,stroke-width:2px
    style D fill:#bbf,stroke:#333,stroke-width:2px
````

1. **The Backend (Private):** We process raw data (STLs, Images, Text) using Python & NumPy to generate optimized brick structures.
2. **The Data Handoff:** The backend exports a lightweight JSON definition.
3. **The Frontend (This Repo):** The web client parses the JSON and reconstructs the model using Three.js geometry for user interaction.
